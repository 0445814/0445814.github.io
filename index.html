[NuxtJS] NuxtJS 教學(0) - Nuxt 是什麼?

發表於 2020-04-01 更新於 2020-04-02 分類於 NuxtJS

前言

使用 Vue Cli 4 完成電商作業後，當然要繼續學習新東西了。

開始學習 Nuxt 後，發現這個框架有許多特性非常好，以後學習重心應該會從 Vue Cli 轉向 Nuxt 了。

總之，這個 Nuxt 系列文會記錄我學習的過程以及踩到的一些雷，希望也能幫助到其他學習者。

Nuxt 是什麼?

官方: Nuxt 是一個基於 Vue.js 的通用應用框架。
Nuxt 是一個高度封裝的 SSR 應用框架。

所以，如果本身就是寫 Vue 的開發者來說，使用 Nuxt 開發，你仍然是寫 Vue，並沒有不同，只是 Nuxt 多了許多特色，

雖然官網介紹已經將特色寫得很清楚，這裡只列出我認為與 Vue Cli 最明顯差別，例如:

  • 伺服器端渲染
  • 快速建立路由
  • 方便的 nuxt.config.js 管理

伺服器端渲染

如果曾經使用過 Vue Cli 開發，就會發現如果對網站按右鍵 -> 檢查網頁原始碼，只會看到一些基本標籤，沒辦法看到網站內容，

例如查看一個 SPA 網站原始碼:

這是因為 SPA (Single page application) 單頁面應用程式是使用 JavaScript 進行前端渲染。一般來說通常是前端使用 AJAX 對伺服器發送請求，伺服器回應數據後，網頁的 JS 會根據數據渲染前端 UI 介面，所有東西都是 JS 動態產生的，所以打開原始碼會看不到內容。這會導致搜尋引擎爬蟲只會看到你的網頁內容是一片空白，對 SEO 相當不好。

但如果是使用 Nuxt 時:


Nuxt 在使用者一開始進入網頁時，會在後端先將頁面生好，再傳給瀏覽器給使用者，所以打開網頁原始碼是可以看見內容的，這就是 SSR (Server side render) 伺服器端渲染

並且之後使用者在瀏覽網站時，例如跳轉路由，仍然是 SPA 體驗，意即同時擁有了 SSR 渲染對 SEO 友善的好處，也保留了 SPA 良好的使用者體驗。

感覺之後可以額外寫一篇關於 前端渲染、後端渲染與前後端分離的文章。

快速建立路由

在 Vue Cli 建立路由時需要自己配置(當然建立專案時直接安裝 vue-router，另當別論 XD)，得建立一個 JS 檔案配置好路由後，在 main.js (進入點) 引入它，雖然稱不上麻煩，但我認為 Nuxt 在路由方面做得更直觀，只需要在 pages 資料夾內建立路由名稱資料夾放置要渲染的 .vue 檔即可，例如新增一個 products 資料夾，裡面放入 index.vue

這樣簡單的建立資料夾，Nuxt 就會自動配置路由，是不是相當方便呢 !
